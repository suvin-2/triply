| 정산 완료된 방에 새 지출 추가 가능 여부 | 불가 | 확정 | 정산 완료 시 FAB 숨김 + "정산이 완료된 여행이에요" 안내 텍스트 표시. 되돌리기 기능으로 재활성화 가능 |
| 방 삭제 기능 | 방장만 가능 (아래 방 삭제 정책 참고) | 확정 | — |

### 방장 권한 정책

- 방 생성 시 `crypto.randomUUID()`로 `ownerToken` 발급
- Firebase `rooms/{roomId}/ownerToken` 에 저장
- localStorage `triply_owner_{roomId}` 에도 저장
- 정산 완료(`status: done`) 상태인 방만 완전 삭제 가능
- 삭제 시 Firebase room 데이터 완전 제거
- 삭제 시 확인 다이얼로그 필수: "삭제하면 모든 데이터가 사라지고 복구할 수 없어요."

### 참여자 숨김 정책

- **정산 완료(status: done) 상태인 방만** 숨김 처리 가능 (확정)
- localStorage `triply_hidden_rooms` 배열에 `roomId` 추가
- 숨긴 방은 홈 목록에서 안 보임
- 숨긴 여행 목록(`/hidden`)에서 복원 가능

### 방장 삭제 시 참여자 처리

- Firebase 데이터 삭제로 조회 불가
- `useRoom.ts`에서 `snapshot.exists()` false 감지 시 localStorage에서 자동 제거
- 홈 목록에서 자연스럽게 사라짐
- 숨긴 방 목록에서도 자동 제거

## 인원 관련

| 정책 | 값 | 상태 | 추천 이유 |
|---|---|---|---|
| 이름 최대 글자 수 | 10자 | 추천값 | 한국 이름(최대 4자) 기준으로 충분. 닉네임·별명 고려해 10자로 여유 확보. ExpenseCard 참여자 표시 UI가 깨지지 않는 한계선 |

| 최소 인원 | 2명 | 확정 | — |
| 이름 중복 허용 여부 | 불가 | 확정 | — |

## 지출 관련

| 정책 | 값 | 상태 | 추천 이유 |
|---|---|---|---|
| 지출 최대 금액 | 10,000,000원 (1천만 원) | 확정 | 숙소·렌터카 등 고액 지출 커버 가능. 커스텀 키패드 자릿수(7자리) 맞춤. 초과 입력 시 키패드에서 자동 차단 (`AddExpenseSheet`, `EditExpenseSheet`) |
| 항목명 최대 글자 수 | 20자 | 추천값 | "제주 저녁 식사 모둠" 수준이면 20자로 충분. ExpenseCard 1줄 표시 기준 |
| 지출 수정 기능 | 가능 (전원, 정산 완료 전에만) | 확정 | 오타·금액 실수 수정 필요. 권한 없는 서비스 특성상 전원 수정 허용. 정산 완료(status: done) 방은 수정 불가 |
| 지출 삭제 권한 | 전원 가능 | 추천값 | 친밀 그룹 신뢰 기반. 삭제 시 확인 다이얼로그 필수 |
| 최대 지출 건수 | 200건/방 | 확정 | 7박 여행 기준 하루 5~10건이면 최대 70건 수준. 200건은 여유 있는 상한선. 초과 시 FAB 클릭 → "지출이 너무 많아요. 최대 200건까지 입력할 수 있어요." 토스트 표시 (`TripScreen`) |

## 정산 관련

| 정책 | 값 | 상태 | 추천 이유 |
|---|---|---|---|
| 정산 완료 후 되돌리기 | 가능 (시간 제한 없음, 확인 다이얼로그만으로 처리) | 확정 | Firebase `status`를 `"active"`로 되돌리는 방식. SettleScreen 하단 "정산 완료 취소하기" 버튼 (`status === 'done'`일 때만 노출) |

| 금액 반올림 기준 | 100원 단위 | 확정 | — |
| 1인당 부담금 계산 방식 | 100원 단위 반올림 (`Math.round(n / 100) * 100`) | 확정 | `src/utils/settlement.ts` 구현과 일치. 소수점 누적으로 인한 무한 루프 방지 목적 |
