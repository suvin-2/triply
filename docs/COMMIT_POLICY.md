# Triply 커밋 메시지 정책

## 형식
[타입] 제목 (50자 이내)

본문 (선택, 72자 이내 줄바꿈)
- 무엇을 왜 변경했는지 설명
- 여러 줄 가능

## 타입 목록
| 타입 | 설명 | 예시 |
|---|---|---|
| [feat] | 새로운 기능 추가 | [feat] 방장 토큰 기반 삭제 기능 추가 |
| [fix] | 버그 수정 | [fix] 한글 IME 입력 시 마지막 글자 중복 추가 오류 수정 |
| [design] | UI/디자인 변경 | [design] 날짜 입력 필드 화면 넘침 수정 |
| [refactor] | 기능 변경 없는 코드 개선 | [refactor] 폼 state 객체로 묶어서 관리 |
| [test] | 테스트 추가/수정 | [test] 정산 알고리즘 엣지케이스 단위 테스트 추가 |
| [docs] | 문서 추가/수정 | [docs] POLICY.md 삭제 정책 업데이트 |
| [chore] | 빌드, 설정, 패키지 변경 | [chore] Playwright MCP 설정 추가 |
| [perf] | 성능 개선 | [perf] Firebase 구독 범위 최소화 |

## 규칙
- 제목은 한국어로 작성
- 제목 끝에 마침표 없음
- 현재형으로 작성 (추가했다 X → 추가 O)
- 타입은 영문 대괄호, 내용은 한국어
- 본문은 선택사항이나 복잡한 변경은 작성 권장

## 나쁜 예 / 좋은 예
나쁜 예:
- 수정함
- fixed bug
- [fix] 버그를 수정했습니다.

좋은 예:
- [fix] 정산 완료 버튼 중복 클릭 방지 처리
- [feat] 숨긴 여행 목록 화면 추가
- [chore] vitest 테스트 환경 설정
